Frequently Asked Questions

Everything you need to know about Opacus

Opacus is an identity system for AI agents. It lets agents prove who they are, control their data, and interact securely across different networks—all using blockchain technology.

A Data Access Consortium is a group of entities that collectively manage access to data streams. DACs can define membership rules, stake requirements, and permission controls through on-chain governance. Each DAC operates independently with its own policies and membership criteria.

H3-DAC provides two production-ready SDKs: TypeScript (for Node.js and browsers) and Rust (for high-performance applications). Both SDKs offer complete functionality including transport layers, cryptography, and smart contract integration.

Agents register on-chain with Ed25519 signing keys and X25519 encryption keys. Authentication uses cryptographic signatures to prove identity. The system supports time-locked key rotation for enhanced security, allowing agents to update their keys while maintaining identity continuity.

The TypeScript SDK supports WebSocket and WebTransport protocols, while the Rust SDK uses QUIC transport with Quinn. All transports use TLS 1.3 for security and support multiplexing for efficient data streaming.

The MsgEscrow contract enables ERC20-based micropayments for data access. Users deposit tokens into escrow, and authorized relayers can settle payments based on usage. This allows for efficient message-level billing without per-message blockchain transactions.

Yes, Opacus is deployed on mainnet with all five core contracts fully operational. Both SDKs are stable and include comprehensive test coverage and documentation.

For TypeScript: Node.js 18+ or modern browsers. For Rust: rustc 1.70+. Smart contract interaction requires an EVM-compatible wallet with sufficient gas tokens for transactions.
